# SDOX to HTML Dictionary

**Version:** 0.1.0  
**Purpose:** This document serves as a standardized reference dictionary for translating Scalable Document (SDOX) syntax into Semantic HTML. This ensures consistency when building parsers or renderers across different platforms, languages, or frameworks.

---

## 📄 Structure Category

### `#document`
The root node of an SDOX document.
**HTML Equivalent:**
```html
<main class="sdox-doc">
  <!-- children -->
</main>
```

### `#title`
Headings with an explicit `level` (1-6).
**HTML Equivalent:**
```html
<h{level} id="{id}" class="sdox-title">
  <!-- content -->
</h{level}>
```

### `#paragraph`
Standard block of text.
**HTML Equivalent:**
```html
<p class="sdox-paragraph">
  <!-- content or children -->
</p>
```

### `#section`
A logical section of the document, optionally with an `id` and `title`.
**HTML Equivalent:**
```html
<section id="{id}" class="sdox-section">
  <!-- if title exists -->
  <h2 class="section-title">{title}</h2>
  
  <div class="section-body">
    <!-- children -->
  </div>
</section>
```

### `#divider`
A horizontal rule or decorative separator.
**HTML Equivalent:**
```html
<hr class="sdox-divider" />
<!-- OR decorative version -->
<div class="sdox-divider"><span>◆</span></div>
```

## ✏️ Text & Semantics Category

### `#text`
Inline styled text.
**Attributes Mapping:**
- `style="bold"` -> `font-weight: 700;` or `<strong>`
- `style="italic"` -> `font-style: italic;` or `<em>`
- `style="underline"` -> `text-decoration: underline;`
- `style="strikethrough"` -> `text-decoration: line-through;` or `<s>`
- `color` -> `color: {color};`
- `size` -> `font-size: {size};`

**HTML Equivalent:**
```html
<span style="font-weight: 700; color: #f87171;">
  <!-- content -->
</span>
```

### `#quote`
A blockquote, optionally with an `author`.
**HTML Equivalent:**
```html
<blockquote class="sdox-quote">
  <div class="quote-body">
    <!-- content -->
    <!-- if author exists -->
    <footer class="quote-author">— {author}</footer>
  </div>
</blockquote>
```

### `#note`
A callout box for important notices. Types: `info`, `warning`, `danger`, `success`.
**HTML Equivalent:**
```html
<div class="sdox-note note-{type}">
  <div class="note-sidebar">
    <!-- Icon based on type (e.g. ⚠️ for warning) -->
  </div>
  <div class="note-content">
    <div class="note-type">{type}</div>
    <div class="note-body">
      <!-- content or children -->
    </div>
  </div>
</div>
```

## 📋 Lists Category

### `#list`
List containers based on `type`.
**HTML Equivalent:**
- `type="unordered"` -> `<ul class="sdox-list list-unordered">`
- `type="ordered"` -> `<ol class="sdox-list list-ordered">`
- `type="checklist"` -> `<ul class="sdox-list list-checklist">`

### `#item`
A single list item. If it belongs to a checklist, it includes a visual indicator based on `done`.
**HTML Equivalent (Unordered/Ordered):**
```html
<li class="sdox-item">
  <span class="item-text"><!-- content --></span>
</li>
```
**HTML Equivalent (Checklist):**
```html
<li class="sdox-item {done ? 'done' : ''}">
  <span class="check-box {done ? 'checked' : ''}">✓</span>
  <span class="item-text"><!-- content --></span>
</li>
```

## 🔗 Links & Media Category

### `#url`
Hyperlink.
**HTML Equivalent:**
```html
<a href="{href}" target="{target}" title="{title}" class="sdox-url">
  <!-- content -->
</a>
```

### `#image`
An image with accessibility alt text and optional dimensions. Should be wrapped in a figure for captions.
**HTML Equivalent:**
```html
<figure class="sdox-figure">
  <img src="{src}" alt="{alt}" width="{width}" height="{height}" />
  <!-- if alt is provided -->
  <figcaption>{alt}</figcaption>
</figure>
```

## 💻 Code Category

### `#code`
A preformatted code block. Should include syntax highlighting based on `language`.
**HTML Equivalent:**
```html
<div class="sdox-code-wrap">
  <!-- if title or language is provided -->
  <div class="code-header">
    <span class="code-title">{title}</span>
    <span class="code-lang">{language}</span>
  </div>
  <div class="code-body">
    <pre><code class="language-{language}">
      <!-- content -->
    </code></pre>
  </div>
</div>
```

### `#output`
Command output, terminal responses, etc.
**HTML Equivalent:**
```html
<div class="sdox-output">
  <div class="output-header">
    <span class="output-label">{type}</span>
  </div>
  <pre class="output-body">
    <!-- content -->
  </pre>
</div>
```

---

## 📊 Tables Category

### `#table`, `#row`, `#cell`
Standard tabular data mapped directly to HTML table elements.
**HTML Equivalent:**
```html
<div class="sdox-table-wrap">
  <table class="sdox-table">
    <tr class="sdox-row">
      <!-- if header=true -->
      <th class="sdox-th"><!-- content --></th>
      <!-- else -->
      <td class="sdox-td"><!-- content --></td>
    </tr>
  </table>
</div>
```

## 🧩 Components & Layout Category

### `#card`
A self-contained block.
**HTML Equivalent:**
```html
<div class="sdox-card">
  <!-- if title -->
  <div class="card-header">{title}</div>
  <div class="card-body">
    <!-- content -->
  </div>
</div>
```

### `#tab`
A tabbed panel UI.
**HTML Equivalent:**
```html
<div class="sdox-tab">
  <div class="tab-bar">
    <span class="tab-active">{title}</span>
  </div>
  <div class="tab-content">
    <!-- content -->
  </div>
</div>
```

### `#accordion`
A collapsible section. Native HTML `<details>` and `<summary>` are highly recommended if not using a JS framework.
**HTML Equivalent (Native):**
```html
<details class="sdox-accordion">
  <summary class="accordion-toggle">{title}</summary>
  <div class="accordion-body">
    <!-- content -->
  </div>
</details>
```

## 📦 Modularization Category

*Note: In an actual production compiler, these tags might be resolved server-side and completely replaced by their referenced content. If rendered for preview/documentation, they are represented as chips or containers.*

### `#include` & `#use`
**HTML Equivalent:**
```html
<div class="sdox-chip">
  <code>{src or template}</code>
</div>
```

### `#template`
**HTML Equivalent:**
```html
<div class="sdox-template">
  <div class="template-header">{name}</div>
  <div class="template-body">
    <!-- content -->
  </div>
</div>
```

## 🤖 AI-Native Category

All AI tags (`#dataset`, `#instruction`, `#response`, `#example`, `#metadata`, `#embedding`, `#chunk`, `#context`, `#completion`) share a unified visual block structure but maintain distinct semantic meaning. 

### Unified AI Block Structure
**HTML Equivalent:**
```html
<div class="sdox-ai-block ai-{tagName}">
  <div class="ai-header">
    <span class="ai-icon"><!-- Tag Specific Icon --></span>
    <span class="ai-label">{tagName}</span>
    
    <!-- Optional badges based on attributes -->
    <span class="badge priority-badge">{priority}</span>
    <span class="badge model-badge">{model}</span>
    <span class="badge scope-badge">{scope}</span>
  </div>
  
  <div class="ai-body">
    <!-- content -->
  </div>
</div>
```

### `#metadata` (Special Case)
Metadata renders its children (`#item`) as a grid of key-value pairs rather than standard body text.
**HTML Equivalent:**
```html
<div class="sdox-ai-block ai-metadata">
  <div class="ai-header">Metadata</div>
  <div class="ai-body">
    <div class="metadata-grid">
      <!-- For each #item -->
      <div class="meta-row">
        <div class="meta-key">{key}</div>
        <div class="meta-val">{content}</div>
      </div>
    </div>
  </div>
</div>
```
